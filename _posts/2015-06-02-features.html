Documented User Scenarios with Screenshots

Inside Scenarioo you can document the important test scenarios of your application with UI screenshots and arbitrary additional information, generated out of your automated UI tests. Scenarioo lets you jump, allmost like a kangaroo, through the steps and screenshots of the tested user scenarios of your application's use cases.

Always up to date

Scenarioo is a living documentation that is generated and updated every night, when you run your automated UI tests on your build server. Scenarioo turns your UI tests into a functional documentation of your software that is allways in synch with what is currently implemented.

Accessible for all

All people invollved in the development of your software can use the Scenarioo webapplication to browse comfortably inside the generated documentation. This is a common source of information and basis for collaboration for product owners, business analysts, product managers, requirements engineers, testers, supporters, developers and operators.

UI testing toolkit independent

Scenarioo provides a documentation format that is independent of your testing technology. It is possible to write Scenarioo XML files and screenshots from arbitrary UI testing toolkits. For C#, Java and JavaScript there is even a scenarioo docu writer library that you can use for writing the docu in your tests.
